Common Mistakes to Avoid

Transmission vs Propagation

Mistake: Thinking Transmission delay depends on distance.

Fact: Transmission (L/R) depends on Packet Size and Link Speed. Propagation (d/s) depends on Distance.

Routing vs Forwarding

Mistake: Using the terms interchangeably.

Fact: Routing is the Global process of finding paths. Forwarding is the Local process of moving packets between ports.

RIP Metric

Mistake: Forgetting that 16 hops means unreachable.

Fact: RIP's maximum valid hop count is 15. 16 is "infinity".

VLAN Access vs Trunk

Mistake: Putting a tag on an Access port frame.

Fact: Access ports frames are Untagged. Trunk ports frames are Tagged (802.1Q).

Packet vs Circuit Switching

Mistake: Thinking Packet Switching has dedicated resources.

Fact: Circuit Switching has Reserved bandwidth. Packet Switching uses Statistical Multiplexing.

Packet Tracer Labs

Lab 1: Static vs Dynamic (RIP)

Topology: 3 Routers in a triangle.

  • Config: router rip, version 2, network ...
  • Behavior: Verify path change when one link fails.
  • Mistake: Forgetting no auto-summary.

Lab 2: OSPF Multi-Area

Topology: Area 0 (Backbone) + Area 1.

  • Config: network [net] [wildcard] area [id]
  • Behavior: Check show ip ospf database for Inter-Area routes.
  • Mistake: Mismatched Area IDs on a single link.

Lab 3: VLAN Trunking & SVI

Topology: 2 Switches, 2 VLANs, 1 Router-on-a-Stick.

  • Config: encapsulation dot1Q [vlan] on sub-interfaces.
  • Behavior: Ping between different VLANs via the router.
  • Mistake: Forgetting switchport mode trunk on switch-to-switch link.

Cisco CLI (Study Commands)

Router# show ip route
View the IP routing table to see all known networks and their next hops.
Router# show ip ospf neighbor
View OSPF neighbor relationships to verify adjacencies.
Router# show ip ospf database
View the Link-State Database (LSDB) which contains all LSAs known to the router.
Router# show vlan brief
View VLAN assignments and status on a switch.
Router(config)# router rip
Enter RIP configuration mode.
Router(config-router)# version 2
Enable RIP version 2 (supports VLSM and multicast updates).
Router(config-router)# network 192.168.1.0
Enable RIP on the specified network interface.
Router(config-router)# no auto-summary
Disable classful summarization (required for VLSM).
Router(config)# router ospf 1
Enter OSPF configuration mode with process ID 1.
Router(config-router)# router-id 1.1.1.1
Set a unique 32-bit identifier for the OSPF router.
Router(config-router)# network 10.0.0.0 0.0.0.255 area 0
Enable OSPF on interfaces matching the network/wildcard mask in Area 0.
Router(config-router)# default-information originate
Advertise a default route to other OSPF routers.
Switch(config)# vlan 10
Create VLAN 10 and enter VLAN configuration mode.
Switch(config-vlan)# name Engineering
Assign a descriptive name to the VLAN.
Switch(config-if)# switchport mode access
Set the interface to access mode (for end devices).
Switch(config-if)# switchport access vlan 10
Assign the interface to VLAN 10.
Switch(config-if)# switchport mode trunk
Set the interface to trunk mode (for switch-to-switch links).
